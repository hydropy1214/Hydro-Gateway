# Build HYDROPY - Complete Android SMS Gateway Platform

## IMPORTANT

Build a complete working product.

Do not build only a demo.

The final project must include:

1. Web dashboard
2. Backend server
3. Database
4. Native Android Gateway APK source code
5. APK build output
6. Complete documentation

The system should be easy to install and operate.

---

# PRODUCT PURPOSE

HYDROPY is a private SMS gateway management platform.

The web dashboard is the main control center.

Android phones connect to the platform and act as SMS gateway devices.

The user installs the HYDROPY Gateway APK on Android phones, connects them once, and then manages everything from the dashboard.

---

# IMPORTANT MOBILE REQUIREMENT

DO NOT USE:

* Expo
* React Native
* Flutter
* Ionic
* Capacitor

The Android application must be:

* Native Kotlin Android
* Android Studio compatible
* Gradle project

The repository must contain:

```

```

with a complete Android Studio project.

The project must be capable of generating:

```
HYDROPY-Gateway.apk
```

The APK should be ready for installation.

# USER EXPERIENCE GOAL

The whole system should be simple:

## Step 1

Open dashboard.

## Step 2

Download APK from dashboard.

Example:

```
Devices
   |
   |
Download Gateway APK
```

## Step 3

Install APK on Android phone.

## Step 4

Open APK.

The app shows:

```
HYDROPY Gateway

Server:
https://your-server.com

Pair Code:

XXXXXX

[CONNECT DEVICE]
```

## Step 5

Device connects.

Dashboard shows:

```
Device Online

Ready for SMS
```

After that no regular interaction should be required.
